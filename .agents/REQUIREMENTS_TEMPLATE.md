## 9. 継続的改善（フェーズ3）

> **フィードバック駆動の自動改善を有効にするか確認**

### 9.1 フィードバック機能

| 項目 | 選択 |
|------|------|
| フィードバック収集ページを含めるか | ✅ はい / ❌ いいえ |
| 自動 PR 作成を有効にするか | ✅ はい / ❌ いいえ |

**「はい」の場合、以下を実装**:

- `/feedback` ページ（ユーザーからのフィードバック投稿）
- フィードバック分類・優先度判定
- GitHub Issue 自動作成
- 条件を満たす場合の自動 PR 作成

### 9.2 フィードバック設定

フィードバック機能の挙動は、使用する AI ツールの設定ファイルに反映する（Codex CLI の場合 `.codex/feedback_config.toml`、Claude Code の場合 `.claude/settings.json` のメタデータ or プロジェクト独自設定）。

```toml
# 例: Codex CLI — .codex/feedback_config.toml
[feedback]
enabled = true/false
auto_implement = true/false
auto_implement_max_priority = "low"/"medium"/"high"
```

### 9.3 フィードバックカテゴリ

| カテゴリ | 説明 | 自動実装対象 |
|---------|------|-------------|
| バグ報告 | 動作不具合の報告 | 軽微なもののみ |
| 機能要望 | 新機能のリクエスト | ❌ Issue のみ |
| 改善提案 | 既存機能の改善 | ✅ 条件付き |
| その他 | 上記以外 | ❌ Issue のみ |
